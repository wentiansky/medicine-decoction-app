# 熬中药计时器

熬中药计时器是一款面向中药煎煮流程的移动计时工具。应用会按照泡水、三轮大火、小火的完整流程进行倒计时，并在阶段结束时通过系统通知、浮窗提醒、响铃和振动提示用户，适合用户把应用切到后台后继续等待提醒。

## 功能亮点

- **分阶段熬药流程**：内置泡水、第一次煎药、第二次煎药、第三次煎药流程，每轮煎药包含大火和小火两个阶段。
- **自定义计时时长**：支持配置泡水时间、大火时间和小火时间，配置会自动保存到本地。
- **小数分钟测试值**：支持输入 `0.1`、`0.11` 这类小数分钟，配置页提供一键填入测试值，方便快速验证提醒链路。
- **后台准时提醒**：Android 端使用原生精确闹钟能力触发阶段完成提醒，应用切到后台后也能收到提醒。
- **通知栏提醒**：阶段完成后会发送系统通知，点击通知会回到应用，不会额外触发闹钟页面。
- **浮窗闹钟提醒**：在获得“显示在其他应用上层”权限后，提醒可以覆盖当前正在使用的应用，并伴随响铃和振动。
- **锁屏与息屏场景处理**：针对锁屏、息屏等场景增加原生闹钟页面和亮屏提醒策略，提升用户错过提醒的概率。
- **渐进式权限引导**：权限弹窗按场景解释，后台/其他应用场景引导开启悬浮窗，锁屏场景提示按需开启后台弹出界面和锁屏显示。
- **主页常驻权限入口**：首页保留轻量的一行权限入口，用户后续想补充锁屏相关权限时可以随时进入。
- **Android 返回手势优化**：返回手势会先关闭弹窗，再从子页面返回主页，主页连续两次返回才退出应用。
- **阶段手动继续**：每个阶段结束后，下一个阶段默认处于暂停等待状态，需要用户点击继续后才会开始计时。
- **清晰的完成状态**：全部阶段结束后展示完成页面，不会突然回到首页。
- **运行日志页面**：应用内提供日志入口，记录权限检查、闹钟调度、原生回调、应用前后台切换等关键事件。
- **日志分页与多选复制**：日志列表按页展示，支持加载更多、多选复制，便于真机测试后快速反馈问题。

## 熬药流程

默认流程共 7 个计时阶段：

1. 泡水
2. 第一次煎药：大火
3. 第一次煎药：小火
4. 第二次煎药：大火
5. 第二次煎药：小火
6. 第三次煎药：大火
7. 第三次煎药：小火

每个阶段完成后都会进入提醒状态。用户确认提醒并回到应用后，可以根据页面提示手动开始下一个阶段。

## 提醒能力

应用目前主要围绕 Android 真机体验做了原生增强：

- 前台运行时，阶段到期会优先请求原生闹钟呈现；如果原生呈现失败，再退回应用内提醒。
- 后台运行时，阶段到期会由原生闹钟触发通知、响铃、振动，并尽量展示锁屏页面或浮窗提醒。
- 通知栏始终保留阶段完成通知，作为用户返回应用的入口。
- 如果没有授予浮窗权限，应用仍会尽量通过通知栏和应用内兜底弹窗提示用户。
- 不同 Android 厂商会对后台弹窗、悬浮窗和自启动做额外限制，实际入口可能会因系统版本和厂商定制而不同。

## 权限说明

为了让后台提醒尽可能可靠，Android 端会引导用户开启以下权限：

- **通知权限**：用于显示阶段完成通知。
- **显示在其他应用上层/悬浮窗**：用于切到后台或浏览其他应用时展示浮窗提醒。
- **后台弹出界面**：部分厂商系统用于允许锁屏或后台场景弹出提醒界面。
- **锁屏显示**：部分厂商系统用于允许提醒显示在锁屏层级。

应用会在页面中提供权限状态和开启入口，并尽量跳转到更接近当前应用的系统设置页。由于 MIUI、ColorOS、OriginOS、HarmonyOS 等系统的权限页面差异较大，后台弹出界面和锁屏显示这类权限不一定能被系统 API 稳定检测，因此应用只做场景说明和入口保留，不再要求用户手动勾选确认。

## 使用说明

### 配置时间

1. 打开应用首页。
2. 进入配置时间页面。
3. 修改泡水时间、大火时间和小火时间。
4. 测试时可以点击测试值按钮，一键填入 `0.11` 分钟。
5. 保存后返回首页开始熬药。

### 开始熬药

1. 点击开始按钮进入第一个阶段。
2. 阶段倒计时结束后，根据提醒完成当前操作。
3. 回到应用后，确认下一个阶段时间，点击继续开始计时。
4. 重复直到全部阶段完成。

### 查看日志

1. 点击首页日志入口进入运行日志页面。
2. 查看权限、闹钟调度、通知、浮窗、前后台切换等日志。
3. 日志较多时可以点击“加载更多日志”继续查看。
4. 需要反馈问题时，可以多选相关日志并复制。

## 安装与运行

### 前置条件

- Node.js 20+
- npm
- Android Studio 或可用的 Android 真机/模拟器

### 安装依赖

```bash
npm install
```

### 启动开发服务

```bash
npm start
```

### 运行 Android

```bash
npm run android
```

### 运行测试

```bash
npm test
```

## 注意事项

- 当前提醒能力重点适配 Android 真机，iOS 和 Web 不具备同等的原生后台闹钟能力。
- 浮窗提醒依赖系统权限，用户未开启时会退化为通知栏提醒和应用内提醒。
- Android 厂商系统对后台行为限制不同，真机验证时建议同时查看应用内日志。
- 如需打 Android 包，请先明确提出打包需求；日常开发改动默认只做轻量验证。
