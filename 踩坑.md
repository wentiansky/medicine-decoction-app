# 熬中药计时器踩坑记录

## 1. 只弹出应用内到期弹窗，未弹出系统闹钟提醒

### 遇到的问题

计时器倒计时结束后，只有 App 内部的 `Alert` 弹窗提示。只要用户不盯着 App，或者 App 进入后台，就很容易错过熬药阶段完成提醒。

### 原因

最初的提醒逻辑主要依赖 JS 层计时和应用内弹窗。应用内弹窗只能在 App 前台可见时发挥作用，不能作为系统级提醒，也无法保证在后台准时唤醒用户。

另外，Expo/React Native 的普通通知能力可以作为提醒兜底，但在 Android 真机上，普通通知不等于“闹钟级提醒”。如果需要更接近系统闹钟的准时触发，需要进入 Android 原生层使用 `AlarmManager`。

### 解决方案

新增 Android 原生闹钟模块：

- 使用 `AlarmManager.setAlarmClock` 调度阶段到期提醒。
- 到点后通过原生广播接收器处理提醒。
- 创建高优先级通知渠道，保留通知栏提醒。
- 增加原生日志，方便从 App 的日志页看到调度、广播、通知、闹钟页面/浮窗是否执行。

同时保留 JS 层的应用内提示作为前台兜底，但不再把它当作后台提醒的主路径。

### 取舍和考量

- `AlarmManager` 比 JS 定时器可靠，适合熬药这种不能错过的场景。
- Android 精确闹钟权限需要用户授权，体验上会多一步，但这是准时提醒的必要成本。
- 普通通知继续保留，因为它是浮窗失败或用户错过浮窗时的重要兜底。

## 2. App 内能弹系统闹钟，切后台后不能弹出系统闹钟提醒

### 遇到的问题

在 App 内部测试时，闹钟提醒能够出现。但切到后台，或者用户正在使用其他 App 时，到点后只看到通知栏通知，没有全局弹出闹钟提醒。

日志显示：

- 原生闹钟已经成功调度。
- 广播已经准时收到。
- 通知已经成功发送。
- 从广播里请求启动闹钟 Activity 也执行了。
- 但真正的提醒页面没有覆盖到当前正在使用的 App 上方。

### 原因

Android 对“后台启动 Activity”限制越来越严格。即使 App 有精确闹钟权限、通知权限和全屏提醒能力，不同 Android 版本和不同厂商 ROM 仍可能拦截后台 Activity 直接覆盖当前界面。

也就是说，问题不是闹钟没触发，而是“从后台把一个普通 Activity 拉到用户当前正在使用的 App 上方”不可靠。

### 解决方案

把“系统闹钟页面”方案调整为“浮窗闹钟”方案：

- 新增 `SYSTEM_ALERT_WINDOW` 权限，也就是“显示在其他应用上层”。
- 到点后由原生广播启动 `AlarmOverlayService`。
- `AlarmOverlayService` 使用浮窗覆盖当前界面，显示闹钟提醒。
- 浮窗服务播放闹钟声、持续震动，并提供“我知道了”和“打开应用”按钮。
- 通知栏通知仍然保留，但点击通知只回到 App，不再触发闹钟提醒页。

### 取舍和考量

- “显示在其他应用上层”权限对用户来说比较敏感，但它正是实现“正在使用其他 App 时也弹出提醒”的关键权限。
- 全屏提醒权限在测试机上表现不稳定，所以不再作为必需权限。
- 浮窗方案比后台拉 Activity 更符合当前 Android 的限制模型，也更可控。
- 通知栏通知只做入口和兜底，真正的闹钟提醒由浮窗负责，职责更清晰。

## 3. 权限交互优化

### 遇到的问题

早期版本把缺失权限直接铺在首页卡片里，用户需要手动理解每个权限的含义，再逐个去系统设置里打开。这个体验偏调试工具，不像面向普通用户的主流 App。

特别是 Android 的一些关键权限无法通过一个普通系统弹窗直接授权，例如：

- 精确闹钟权限。
- 显示在其他应用上层权限。

用户必须跳转到系统设置页手动开启。如果 App 只是简单列出权限，用户很容易觉得麻烦，甚至直接放弃使用。

### 原因

Android 权限分两类：

- 通知权限这类可以通过系统授权弹窗完成。
- 精确闹钟、显示在其他应用上层这类高风险权限，必须跳到系统设置页，由用户手动开启。

因此，不能完全做成“一键同意后全部正常”，只能把流程包装得更清楚、更短、更像引导。

### 解决方案

新增权限引导交互：

- 缺权限时弹出“开启可靠提醒”引导弹窗。
- 文案使用通用描述，不绑定微信，也不强调全屏提醒。
- 引导只展示当前需要处理的权限，避免一次性把所有设置项甩给用户。
- 用户从系统设置页返回 App 后，自动重新检测权限。
- 如果还有下一项缺失权限，继续引导下一步。
- 首页只保留轻量入口“设置提醒权限”，不再铺一堆权限项。
- 开始/重新开始计时时，如果权限不足，也进入同一套引导流程。

### 取舍和考量

- 无法绕过 Android 对特殊权限的系统设置限制，只能优化解释和步骤。
- 不再强制要求全屏提醒权限，减少用户需要开启的权限数量。
- 必需权限收敛为通知、精确闹钟、显示在其他应用上层。
- 首次使用会多一次引导，但换来的是后台提醒的可靠性。
- 首页保持轻量，权限问题只在必要时出现，降低用户心理负担。
